# Demo Idea Generation Workflow

## 1. Purpose

Generate buildable demo ideas from trends, papers, GitHub projects, product releases, or user signals.

## 2. When to Use

Use when a trend looks promising and needs a small experiment before becoming a PRD or MVP.

## 3. Inputs

- Inspiration source
- User problem
- Target user
- AI capability
- Data / API availability
- Build constraints
- Product goal

## 4. Steps

1. Identify inspiration source and evidence.
2. Define user problem and target user.
3. Define core experience.
4. Identify AI capability and data / API needed.
5. Define key screens and MVP Flow.
6. Suggest tech stack.
7. Define success criteria, risks, and build plan.
8. Recommend sync target: PM Strategy, Design Prototype, or Engineering Build.

## 5. Outputs

- Demo name
- Inspiration source
- User problem
- Core experience
- AI capability
- Data / API needed
- Key screens
- MVP Flow
- Tech stack suggestion
- Success criteria
- Risks
- Build plan

## 6. Quality Checklist

- Demo tests a real product hypothesis.
- Scope is small enough to build.
- AI capability is specific.
- Data / API needs are realistic.
- Next workspace recommendation is clear.

## 7. Common Mistakes

- Making a demo because the technology is new.
- Missing user problem.
- Requiring unavailable data.
- Designing a full product instead of a demo.
- Skipping success criteria.
