# Product Opportunity Extraction Workflow

## 1. Purpose

Extract product opportunities from trend signals and decide whether they are worth exploring.

## 2. When to Use

Use after daily radar, weekly radar, topic research, YouTube research, GitHub research, or paper analysis.

## 3. Inputs

- Source Signal
- User Problem
- Target User
- Scenario
- Product domain
- Technical constraints
- Business constraints

## 4. Steps

1. Identify source signal and evidence quality.
2. Extract the user problem behind the signal.
3. Define target user and scenario.
4. Identify AI capability needed and data needed.
5. Define MVP scope and non-goals.
6. Assess differentiation, feasibility, risks, user value, and business value.
7. Assign priority and next step.

## 5. Outputs

- Source Signal
- User Problem
- Target User
- Scenario
- AI Capability Needed
- Data Needed
- MVP Scope
- Differentiation
- Feasibility
- Risks
- Priority

## 6. Quality Checklist

- Opportunity is tied to a real user problem.
- Evidence is visible.
- MVP scope is realistic.
- Risk is not hidden.
- Next workspace is clear.

## 7. Common Mistakes

- Treating technology novelty as opportunity.
- Missing target user.
- Ignoring data requirements.
- Over-scoping MVP.
- Not comparing user value against risk.
