# GitHub Trending Research Workflow

## 1. Purpose

Analyze GitHub Trending or open-source projects for product potential, integration value, risks, and demo ideas.

## 2. When to Use

Use when exploring AI open-source tools, frameworks, models, agents, RAG systems, coding tools, multimodal tools, or developer infrastructure.

## 3. Inputs

- Topic or repository list
- Timeframe
- Language or tech preference
- Product question
- Integration constraints

## 4. Steps

1. Select repositories by relevance, stars, forks, recent activity, releases, issues, README quality, and examples.
2. Read README for what it does, install path, examples, license, and target users.
3. Check stars, forks, recent activity, issues, releases, and maintainer responsiveness.
4. Identify technical architecture and integration surface.
5. Judge project maturity, productization potential, and risk.
6. Generate integration suggestions and demo ideas.

## 5. Outputs

- Repository analysis table
- README quality notes
- Activity and maturity assessment
- Product potential
- Integration potential
- Risks
- Demo ideas
- Recommended actions

## 6. Quality Checklist

- Stars are not the only signal.
- Recent activity and issues are checked.
- README and examples are evaluated.
- Integration risk is explicit.
- Demo idea is practical.

## 7. Common Mistakes

- Equating stars with quality.
- Ignoring license and maintenance.
- Missing integration complexity.
- Ignoring security or dependency risk.
- Creating demos that require unrealistic infrastructure.
