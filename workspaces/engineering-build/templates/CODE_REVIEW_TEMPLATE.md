# Code Review Template

## 1. Review Goal

Describe what behavior, implementation, or risk this review should validate.

## 2. Related Requirement

Link or summarize the PRD, prototype, API spec, ticket, or acceptance criteria.

## 3. Changed Files

## 4. Product Behavior Check

Check whether the implementation matches PRD, prototype, user flow, and acceptance criteria.

## 5. Code Quality Check

Check readability, maintainability, complexity, duplication, naming, error handling, and architecture fit.

## 6. API / Data Check

Check request / response shape, validation, error codes, compatibility, migrations, indexes, and data integrity.

## 7. Security and Privacy Check

Check auth, permission, sensitive data exposure, injection risk, logging, secrets, privacy, and abuse cases.

## 8. Performance Check

Check latency, database queries, caching, pagination, concurrency, and resource usage.

## 9. Test Coverage Check

Check unit, integration, API, UI, permission, regression, and AI-specific tests.

## 10. UI / UX Consistency Check

Check visual and interaction consistency with design handoff, responsive behavior, and state handling.

## 11. AI Behavior Check

Check output format, prompt safety, fallback, RAG behavior, tool failures, logs, evaluation metrics, cost, and latency.
