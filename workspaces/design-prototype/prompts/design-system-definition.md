# Design System Definition Prompt

## 1. Prompt Name

Design System Definition Prompt

## 2. Use Case

用于为产品或项目定义轻量设计系统。

## 3. Copyable Prompt

```text
请作为 AI Product Builder 的 Design Prototype Partner，为下面的项目定义设计系统。

默认中文输出。请先判断是否已有品牌关键词、目标用户、平台、视觉方向和页面类型；缺失信息列为待确认问题，不要编造。

请输出：
1. 品牌关键词
2. 视觉原则
3. 色彩 token
4. 字体 token
5. 间距 token
6. 圆角 token
7. 阴影 token
8. 图标风格
9. 组件规范
10. 状态规范
11. 动效规范
12. 可访问性规范
13. Do / Don’t
14. 工程实现注意事项
15. 下一步行动建议

项目信息：
[粘贴产品定位、平台、视觉偏好、页面类型和约束]
```

## 4. Expected Output

- 轻量设计系统
- 设计 token 表
- 组件和状态规范
- 动效与可访问性规则
- 工程实现注意事项
