# UI/UX Design Review Workflow

## 1. Purpose

Review an existing UI or screenshot for product clarity, usability, visual quality, accessibility, and implementation readiness.

## 2. When to Use

Use when evaluating a page screenshot, prototype, implemented UI, or redesign direction.

## 3. Inputs

- Screenshot or page description
- Product goal
- Target user
- Platform
- Key user action
- Known design concerns

## 4. Steps

1. Confirm the page goal and primary user action.
2. Review information architecture and content priority.
3. Review visual hierarchy, page layout, alignment, whitespace, and information density.
4. Review color system, typography hierarchy, components, and consistency.
5. Review interaction feedback, component states, and empty / loading / error states.
6. Review mobile adaptation, usability, accessibility, and product goal fit.
7. Make “高级感” concrete by explaining what feels polished or unpolished in color, type, spacing, radius, shadow, icon, state, and motion.
8. Prioritize issues by severity and provide fix suggestions.

## 5. Outputs

- Overall impression
- Issue table with severity, evidence, impact, and fix suggestion
- Prioritized improvement plan
- Mobile and accessibility notes
- Design risks and next step

## 6. Quality Checklist

- Feedback is specific and evidence-based.
- Issues include impact and fix suggestion.
- Visual quality is explained in concrete design terms.
- Component states and edge states are checked.
- Review considers product goal and platform.

## 7. Common Mistakes

- Saying “looks good” without evidence.
- Treating visual polish as personal taste only.
- Ignoring empty, loading, and error states.
- Ignoring mobile adaptation.
- Giving fixes that conflict with the product goal.
