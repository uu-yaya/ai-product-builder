# User Journey Mapping Workflow

## 1. Purpose

Map the user's end-to-end experience so product decisions are grounded in real user intent, emotion, pain points, and touchpoints.

## 2. When to Use

Use before designing key flows, prioritizing features, writing PRD, or reviewing user experience.

## 3. Inputs

- User persona
- Scenario
- Trigger
- Product goal
- Known pain points
- Current journey or alternatives

## 4. Steps

1. Define User Persona and scenario.
2. Identify the trigger that starts the journey.
3. Break the journey into stages.
4. For each stage, list user actions, emotions, pain points, product touchpoints, opportunities, and metrics.
5. Highlight high-friction moments and product opportunities.
6. Translate journey insights into product requirements or prioritization input.

## 5. Outputs

- User Persona
- Scenario
- Trigger
- Journey Stages
- User Actions
- User Emotions
- Pain Points
- Product Touchpoints
- Opportunities
- Metrics

## 6. Quality Checklist

- Journey stages reflect user behavior, not internal team process.
- Emotions and pain points are specific.
- Opportunities connect to product actions.
- Metrics are mapped to journey stages.
- Assumptions are clearly marked.

## 7. Common Mistakes

- Mapping only system steps, not user experience.
- Ignoring emotional friction.
- Creating opportunities that do not solve pain points.
- Skipping metrics.

## 8. Example Command

请为这个目标用户和场景生成用户旅程地图，包含情绪、痛点、产品触点、机会和指标。
