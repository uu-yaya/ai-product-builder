# Terminology

| Term | 中文解释 | Product Meaning | Technical Meaning | Example |
| --- | --- | --- | --- | --- |
| AI Product Builder | AI 产品构建系统 | 帮助从想法到产品落地的工作体系 | 由规则、上下文、模板和工作流组成的协作环境 | 用 AI Product Builder 完成从 PRD 到 MVP 的推进 |
| PM Strategy | 产品策略工作区 | 负责需求、定位、竞品和任务拆解 | 输出结构化产品文档与工程输入 | 在 PM Strategy 中澄清用户场景 |
| Design Prototype | 设计原型工作区 | 负责体验、视觉和原型表达 | 生成 Figma Prompt、页面结构和交互建议 | 为小程序首页生成高保真原型说明 |
| Engineering Build | 工程落地工作区 | 负责代码实现、接口、测试和上线检查 | 阅读代码、修改实现、验证功能 | 根据 PRD 实现 MVP 接口 |
| AI Trend Radar | AI 趋势雷达工作区 | 负责发现趋势、机会和信息差 | 跟踪新闻、论文、GitHub、产品发布 | 汇总最近一周 AI Agent 进展 |
| RAG | 检索增强生成 | 用外部知识提升回答准确性 | 检索文档后把相关上下文交给模型生成 | 基于知识库回答客服问题 |
| Agent | 智能体 | 能自主规划和执行多步任务的 AI 能力 | 模型结合工具调用、记忆和任务循环 | 自动调研竞品并生成报告 |
| Function Calling | 函数调用 | 让 AI 调用确定性工具完成动作 | 模型输出结构化参数调用 API 或函数 | 查询订单状态后生成回复 |
| Workflow | 工作流 | 固定步骤的业务或生产流程 | 多个节点按规则串联执行 | 先检索、再总结、再人工审核 |
| MVP | 最小可行产品 | 用最小成本验证核心价值 | 最小功能集合和技术实现 | 先做一个可用的需求澄清工具 |
| PRD | 产品需求文档 | 描述用户、场景、功能、验收标准 | 工程实现的产品输入 | 写一个登录流程 PRD |
| SRS | 软件需求规格说明 | 将产品需求转为系统需求 | 明确接口、数据、约束和非功能需求 | 定义支付模块的系统需求 |
| High-fidelity Prototype | 高保真原型 | 接近最终体验的视觉和交互方案 | 可用于评审、演示或辅助前端实现 | 生成可交互的 Web 首页原型 |
