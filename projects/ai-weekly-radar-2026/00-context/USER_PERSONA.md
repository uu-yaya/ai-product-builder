## 2. Secondary Personas

简要列出 2–3 个次要用户画像，每个画像同样填上表关键信息。

## 3. User Goals

主要用户希望通过本产品达成的目标。一条一句，最多 5 条。

## 4. Pain Points

主要用户当前遇到的痛点。区分：

- 高频低代价
- 低频高代价
- 工具缺失
- 流程问题
- 信息不透明

## 5. Motivations

驱动用户使用本产品的动机：效率、好奇心、KPI、社交、创作表达、合规要求等。

## 6. Usage Context

- 在什么时间 / 地点 / 任务流中使用
- 与谁协作
- 与哪些其他工具并用
- 中断与回到产品时的状态恢复需求

## 7. Decision Factors

用户在选择 / 持续使用 / 放弃本产品时考虑的因素：

- 学习成本
- 速度
- 输出质量
- 可控性
- 信任 / 隐私 / 合规
- 成本 / 价格

## 8. Success Criteria

从用户视角，"用得好"的判断标准。这与产品指标不同——这是用户自己的口语化判断。
