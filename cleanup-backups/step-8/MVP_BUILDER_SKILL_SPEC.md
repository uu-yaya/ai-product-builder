# mvp-builder Skill Spec

## Skill Name

mvp-builder

## Purpose

Convert product requirements into a small, testable engineering implementation plan.

## Trigger Scenarios

Use when the user wants to build an MVP, create an implementation plan, or connect product requirements to engineering tasks.

## Inputs

- PRD or requirement summary\n- Repo context\n- Tech stack\n- Constraints\n- Target milestone\n- Existing APIs or data models

## Workflow

1. Read project structure before planning implementation.\n2. Map requirements to user flows, APIs, data, and UI changes.\n3. Split work into small milestones.\n4. Define tests and acceptance criteria.\n5. Identify launch and rollback risks.

## Outputs

- Implementation plan\n- Task breakdown\n- API/data notes\n- Test plan\n- Launch checklist

## Quality Checklist

- Plan is small-step and executable.\n- Risks and dependencies are explicit.\n- Tests map to requirements.\n- No production config changes are hidden.\n- MVP avoids unnecessary complexity.

## Guardrails

- Do not code before reading relevant files.\n- Do not over-engineer.\n- Do not modify secrets, production deployment, or auth without explicit risk review.
