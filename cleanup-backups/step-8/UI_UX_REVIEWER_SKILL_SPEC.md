# ui-ux-reviewer Skill Spec

## Skill Name

ui-ux-reviewer

## Purpose

Review product interfaces for usability, hierarchy, clarity, visual quality, and interaction completeness.

## Trigger Scenarios

Use when the user asks for UI/UX critique, page review, design QA, or visual improvement suggestions.

## Inputs

- Screenshot or page description\n- Product goal\n- Target user\n- Device type\n- Brand or style constraints\n- Known problems

## Workflow

1. Inspect information architecture and task flow.\n2. Review visual hierarchy, spacing, alignment, typography, color, states, and motion.\n3. Identify usability and accessibility issues.\n4. Prioritize fixes by impact.\n5. Suggest concrete design improvements.

## Outputs

- UX findings\n- Visual findings\n- Prioritized fixes\n- Improved layout guidance\n- Optional Figma Prompt input

## Quality Checklist

- Findings are specific.\n- Visual issues include why they matter.\n- Mobile behavior is considered.\n- Component states are checked.\n- Recommendations are implementable.

## Guardrails

- Do not merely say the design looks good.\n- Do not copy inspiration references.\n- Do not ignore accessibility or small-screen constraints.
