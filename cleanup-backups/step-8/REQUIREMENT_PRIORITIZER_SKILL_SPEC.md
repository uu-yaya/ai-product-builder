# requirement-prioritizer Skill Spec

## Skill Name

requirement-prioritizer

## Purpose

Rank requirements and define a realistic MVP scope based on value, effort, risk, and learning goals.

## Trigger Scenarios

Use when the user has too many features, needs MVP scope, or wants tradeoff analysis.

## Inputs

- Requirement list\n- Product goal\n- User segments\n- Constraints\n- Technical risks\n- Deadline or resource limits

## Workflow

1. Group requirements by user outcome.\n2. Score value, urgency, effort, risk, and learning potential.\n3. Identify must-have MVP items.\n4. Defer nice-to-have and high-risk non-critical items.\n5. Explain tradeoffs and sequencing.

## Outputs

- Priority table\n- MVP scope\n- Deferred scope\n- Tradeoff rationale\n- Suggested milestones

## Quality Checklist

- Priorities map to user value.\n- MVP is small enough to build.\n- Deferred items are explicit.\n- Risks are not hidden.\n- Sequencing is realistic.

## Guardrails

- Do not prioritize based only on excitement.\n- Do not hide hard tradeoffs.\n- Do not include unvalidated complex AI features in MVP without reason.
