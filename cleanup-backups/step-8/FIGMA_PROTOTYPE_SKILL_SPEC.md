# figma-prototype Skill Spec

## Skill Name

figma-prototype

## Purpose

Create structured Figma Prompts and prototype briefs for high-fidelity product interfaces.

## Trigger Scenarios

Use when the user asks for Figma Prompt, high-fidelity prototype direction, or screen-by-screen interaction design.

## Inputs

- Product goal\n- User journey\n- Screen list\n- Content requirements\n- Visual direction\n- Platform constraints

## Workflow

1. Clarify screen goals and user flow.\n2. Define layout, hierarchy, components, states, and interactions.\n3. Specify visual direction without copying references.\n4. Write a Figma-ready prompt.\n5. Include mobile and responsive notes where relevant.

## Outputs

- Figma Prompt\n- Screen structure\n- Interaction notes\n- Component/state guidance\n- Visual style direction

## Quality Checklist

- Prompt is concrete and buildable.\n- Screens map to user flow.\n- Visual language is coherent.\n- States and responsiveness are included.\n- References are transformed, not copied.

## Guardrails

- Do not generate vague aesthetic prompts.\n- Do not ignore product goals.\n- Do not use copyrighted design language as a direct copy target.
