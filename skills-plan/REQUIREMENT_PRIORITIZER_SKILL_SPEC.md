# requirement-prioritizer Skill Spec

## Skill Name

requirement-prioritizer

## Purpose

Prioritize requirements into a realistic MVP and sequenced roadmap.

## Trigger Scenarios

- The user has too many requirements and needs MVP scope.
- The team needs a clear must-have / should-have / later split.
- Tradeoffs between value, effort, risk, and learning need to be explicit.

## Inputs

- Requirement list
- Product goal
- User segments
- Constraints
- Technical risks
- Deadline or resource limits

## Workflow

1. Group requirements by user outcome.
2. Score value, urgency, effort, risk, and learning potential.
3. Identify must-have MVP items.
4. Defer nice-to-have and high-risk non-critical items.
5. Explain tradeoffs and sequencing.

## Outputs

- Priority table
- MVP scope
- Deferred scope
- Tradeoff rationale
- Suggested milestones

## Quality Checklist

- Priorities map to user value.
- MVP is small enough to build.
- Deferred items are explicit.
- Risks are not hidden.
- Sequencing is realistic.

## Guardrails

- Do not prioritize based only on excitement.
- Do not hide hard tradeoffs.
- Do not include unvalidated complex AI features in MVP without reason.
