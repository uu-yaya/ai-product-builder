# figma-prototype Skill Spec

## Skill Name

figma-prototype

## Purpose

Turn product requirements into concrete Figma-ready prototype prompts and screen structure.

## Trigger Scenarios

- The user asks for a Figma Prompt, high-fidelity prototype prompt, or prototype direction.
- A product idea needs screen structure and interaction states before design execution.
- A design tool needs a concise English prompt grounded in product intent.

## Inputs

- Product goal
- User journey
- Screen list
- Content requirements
- Visual direction
- Platform constraints

## Workflow

1. Clarify screen goals and user flow.
2. Define layout, hierarchy, components, states, and interactions.
3. Specify visual direction without copying references.
4. Write a Figma-ready prompt.
5. Include mobile and responsive notes where relevant.

## Outputs

- Figma Prompt
- Screen structure
- Interaction notes
- Component/state guidance
- Visual style direction

## Quality Checklist

- Prompt is concrete and buildable.
- Screens map to user flow.
- Visual language is coherent.
- States and responsiveness are included.
- References are transformed, not copied.

## Guardrails

- Do not generate vague aesthetic prompts.
- Do not ignore product goals.
- Do not use copyrighted design language as a direct copy target.
