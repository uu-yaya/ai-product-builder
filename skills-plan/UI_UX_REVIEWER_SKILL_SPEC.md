# ui-ux-reviewer Skill Spec

## Skill Name

ui-ux-reviewer

## Purpose

Review product UI and UX with concrete, prioritized improvement guidance.

## Trigger Scenarios

- The user asks for UI review, UX critique, or page improvement advice.
- A screenshot, page implementation, or prototype needs design quality feedback.
- The product needs more specific guidance than vague words like modern or premium.

## Inputs

- Screenshot or page description
- Product goal
- Target user
- Device type
- Brand or style constraints
- Known problems

## Workflow

1. Inspect information architecture and task flow.
2. Review visual hierarchy, spacing, alignment, typography, color, states, and motion.
3. Identify usability and accessibility issues.
4. Prioritize fixes by impact.
5. Suggest concrete design improvements.

## Outputs

- UX findings
- Visual findings
- Prioritized fixes
- Improved layout guidance
- Optional Figma Prompt input

## Quality Checklist

- Findings are specific.
- Visual issues include why they matter.
- Mobile behavior is considered.
- Component states are checked.
- Recommendations are implementable.

## Guardrails

- Do not merely say the design looks good.
- Do not copy inspiration references.
- Do not ignore accessibility or small-screen constraints.
